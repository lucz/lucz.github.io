九評軟件公司一：微軟


我對輪子的好印象完全來自免費門，其他印象都是中性和負面的，「酒瓶」也不例外。不過我覺得他的題目寫得非常好，至少當時我就是看到章節題目而要去看的。不管酒瓶的作者是誰，文章的可信度多低，至少文筆是不錯的，題目也是不錯的。
於是我決定把精華偷過來。至於我文章的可信度和知識性，看來並沒有比酒瓶更高。我沒有內線消息，又是個外行人，當然信用是不存在的了。但是我又是最客觀且最權威的——不在業內，沒有利益衝突，所以客觀；是用戶，而且各種嘗試的經驗還算比較豐富，所以權威。
最客觀、最權威、最低可信度、最缺乏知識性的軟件公司評論文，便呼之欲出了。
然而在發第一篇的時候，我甚至還沒計畫好我的九評要評什麼，甚至九評可能都會被刪減至八評七評六評——多麼不可信任的一個作者啊。
是爲總序。

微軟是影響力最大的公司之一，年淨利過百億美元。一個公司之所以能夠成功，必至少曾有其善良的一面，微軟也不例外。我把早期的微軟叫做 MS ，現在的叫 M$ 。
MS 有兩個不可磨滅的重大功勞：廉價化和簡易化。電腦軟件，在我出生之前，絕對是高科技，絕對是金貴的玩意，不像現在，我這種程序白癡都能寫幾個腳本。所以那時的軟件都是高級人才開發的，開發成本高，而電腦又沒有普及，銷量上不去，所以單價奇高。 MS 則打破了僵局，提供了在當時看來非常便宜的軟件。
從前電腦如此高科技，一方面是硬件問題，但更重要的是軟件問題。那時的電腦就是個黑乎乎的屏幕，當你敲下一堆不知道從哪學來的命令之後，他才乖乖聽話，按你的命令行事。這就太難學了。而 MS 開發了非常簡單好學的圖形界面，識字就能用一些基本功能，門檻低，上手快，把「高科技」帶入千萬家。
到了九十年代，隨著個人電腦的普及， MS 的壟斷地位漸漸鞏固。小時候我還常常聽到 DOS ，也同時聽到蘋果，但到了我真正開始用電腦的時候 (1998) ，我真的只能聽到 Windows 了（ DOS 衹在教科書裏保留一些餘孽，反正那些命令繼續在 Windows 用，學了也不是白學，蘋果卻是教科書都沒有的）。這大概就是 MS 盛極而衰的時候吧。那時很多同學都視 Bill Gates 爲偶像，智慧名聲財富一體的完美形象。

從 Windows 95 到 98 再到 XP ，進步放緩了，但商業運作越來越賣力了， MS 也漸漸變成 M$ ，並成爲被告。
然而作爲盜版用戶，我還是欣然接受這兩次升級的。可 XP 開始，我放棄了系統的升級，也放棄多數了 M$ 軟件的升級。對於 M$ 家的產品，不管是 2006 年的 Vista 還是 2008 年的 Windows 7 （簡稱瘟雞），又或 Office 2007 和 2010 ， IE 7~9 ，都不曾打動我，我「堅守」在五代的陣地上，用的不是 Server 2003 ，就是 XP ，然後配上 Office 2003 和 IE 6 。不是我拒絕新版。我裝過瘟雞，也在學校的電腦上面用過 Office 2007 和 IE 8 ，感覺很慢，就沒升級了。

先說瘟雞。那時我新換一台電腦，於是試了瘟雞。那是 64 位 Ultimate ，最強的版本了。可三天後，我下載了一個新版 GhostXP ，並進行一鍵還原。我承認瘟雞在圖形方面是下了功夫的，可我不需要。我 XP 的介面都不用，而用「經典」主題和開始菜單，也就是 98 風格。 98 風格對我來說已經足夠，有什麼必要改變？而且隨意改變界面還造成用戶不適應的。在 XP 中，你可以設置風格，把 XP 變回 98 的樣子，從而快速適應。而隔了兩代，引擎從 5.1 到 6.1 的瘟雞，居然功能還減少了，不能用老風格了。
而且新版的諸多「優點」對我來說甚至是重大缺點。
第一我需要編輯時視野寬闊，因爲我思維跳躍毫無規律。我會介意瘟雞的任務欄太厚還不能調。而且這任務欄，我實在看不出什麼好來。我一般開的窗口不多（十個以內），瀏覽器的 tab 功能都覺得多餘，就喜歡一鍵到位，直接切換至我需要的窗口。可是瘟雞自作聰明的把同一程序的窗口疊起來了。你那麼喜歡疊，幹嘛不把所有窗口都疊起來？那樣不是更「簡潔」？而且你的圖標做得那麼醜（特別是播放器和文件夾的圖標），小一點也就算了，看不清。偏偏做得那麼肥碩，想看不清到不行……
第二，瘟雞增加了一個蛋疼的功能，那就是如果你把窗口拖動到頂部，就會自動最大化。很不幸，我就是經常拖動窗口來調節的人，而且不想最大化。當然你可能會說瘟雞用瘟雞自帶的拖動到左右就是佔一半不是很好麼。對不起，我想要的不是這個效果。
第三，在盜版如此便利的此時，系統穩定不穩定關我屁事啊。裝 XP 就不用關，裝瘟雞就需要關 UAC ，你不蛋疼把蛋借我疼吧。
第四，資源佔用。 XP 一張 CD 足夠，還帶幾個常用軟件，瘟雞卻是 DVD 級別的。裝起來以後更要多用好幾 GB 的空間。我是資料收集狂，硬盤永遠不夠，還給你？
第五，軟件兼容。已經出來這麼久了，還是有問題，比如曹操傳這個遊戲，現在兼容都不大好，用所謂的 XP 模式都不行。

再說 Office 。
「你祖宗十八代的！」這是我在學校電腦上第一次使用 2007 版時的內心獨白。你升級就升級，功能增加就行了，你幹嘛改風格啊，害得我都找不到常用按鈕了。還有你的製表符能再噁心一點嗎，比正文還清晰，你讓我看文章還是製表符呢？
「你祖宗二十八代的！」這是我在學校電腦上寫完一篇文章帶回宿舍後的內心獨白。該死的 docx 格式，自己家的軟件都不兼容，還要下一個包。你就不能穩定點，或者乾脆改成國際標準 ODF 麼。
「你祖宗三十八代的！」這是我再次嘗試後的內心獨白。什麼時代了，滿街的免費文本編輯軟件都有列模式了，我也都習慣按 Alt 了，你告訴我你沒有？你憑什麼收錢啊？免費的都不如。
「你祖宗八十八代的！」這是我現在的內心獨白。隨便一個文本編輯器，如果沒有文字比對功能，都是我要鄙視的。甚至 Wordpress 一個寫網誌的小程序都能比對了，居然還不能比對？這說明 Office 的代碼非常爛，不像 html 那麼丁是丁卯是卯，所以無法實現比對功能。因爲無法自動比對，用戶只好手工比對，這大概就是 Office 沒有 tab 的原因吧。你 tab 了就不能手工比對了。
一系列嘗試，我覺得 Office 沒有什麼實質上的進展，不值得花時間適應。我繼續和盜版 2003 相依爲命好了。

經過多次教訓，我總算明白了一點，世界上有三種軟件升級，一種叫軟件升級，一種叫中國軟件升級（中國永遠要佔一類），一種叫 M$ 升級。
一般而言，軟件升級時，界面上具有一定的延續性，而功能則不斷增強。界面延續性保證使用者能夠快速適應，而漸漸增加的少量界面調整往往是爲了提供更多的功能，或之前的版本界面還不是很完善，所以稍微增強。而 M$ 升級則是倒過來。功能方面具有很強的延續性，所以用戶閉上眼睛就知道他能做什麼，不能做什麼。但睜開眼睛的時候，還是嚇了一跳：這尼瑪是之前那家公司做的麼？小白見狀更大驚：哇，升級了，趕快換新版。
爲什麼會有這樣奇特的區別？我想了一下，覺得最合理的理由是方式。
很多軟件是按年收費的，比如殺毒軟件，再比如 SuSE Linux Enterprise 。這些軟件就算永遠只用一個版本，也照樣收錢。當然，如果一點進步都沒有，用戶也會跑掉。對於這些公司來說，市場經營的手段在於持續的推出適當更新，而這些更新也主要是修復漏洞和增強功能，這樣用戶覺得物有所值，就會一直購買。
一些軟件基本和硬件綁定，如 MAC OS ，一般用戶不會不買蘋果機吧？硬件更新換代很頻繁，所以這種其實和前面情況差不多。
還有一些小家碧玉軟件，是一次付費終身升級的，就算再也不更新，錢也到手了。當然，如果賣得好，作者會繼續改進，由老客戶推薦給新客戶的。
其實 M$ 的軟件屬於最後那種，比如你買了正版的 XP ，那你確實可以用到海枯石爛。但問題是 M$ 已獲得了 90% 的市場份額，你總不能不讓他再創收了吧？這就是 M$ 軟件弔詭的地方： XP 確實是終身使用了（至少不理會漏洞的小用戶是這樣），但是 XP 到 Vista 不是。
這下明白了吧？爲了創收，只能敦促用戶升級到新的大版本。可是 M$ 沒有新招了啊，怎麼辦？別急，成功從 MS 進化成 M$ 的這家神奇公司有很多招數的。
首先是前面提到的界面大亂改。雖然瞭解了底細之後，我們會覺得這樣的升級意義不大，但這樣確實對小白具有超強的殺傷力。
然後是兼容問題。 MS 非常善於使用「不兼容」來排擠對手，在 DOS 時代就做過這種事。而 M$ 更變態，不但不兼容別家的，甚至自家的都不兼容，讓舊版用戶不舒服——誰叫你不升級，誰不升級我讓誰難受！例子麼，比如 docx ，比如 IE 9 ，比如 DirectX 10 。
再就是停止發佈補丁來威脅。 M$ 不管是樹大招風還是本身問題就多，反正隔三差五就給你報一個重大漏洞什麼的，非要你打補丁。到了要換代的時候更牛逼了： M$ 只要登高一呼：舊版系統將於某年某月某日停止發佈補丁！
最後再來一個大殺器：我不更新，我就是不更新，誰讓你不買新系統？看吧看吧，新出了一個神馬神馬硬件，新系統完美支持的哦，舊系統不支持，你不想換軟件我管不了，你硬件總會壞掉罷？
可是在我看來， M$ 在一定程度上，已經偏離了應有的軌道。
當 M$ 開發一款操作系統時，似乎「革新」是他們的主題詞（我想，蘋果的主題詞一定是「時髦」）。但我想說，對一款系統而言，「效率」「功能」「穩定」「兼容」「便捷」纔是最重要的。如果僅僅是界面的美化和一些小增強，一款舊系統加上一些小軟件就足夠了。 M$ 家的系統並沒有優秀到無可超越，但新系統就是沒有去做應該做的事情。說效率，我沒看見他新系統開機速度有多快，運行時佔用內存和 CPU 有多小。說功能，我沒看見新系統自帶好用的磁盤碎片整理、性能釋放等系統工具，而這些工具對於系統的日常維護是相當重要的。說穩定，新系統也並非沒有藍屏，也需要重裝（我不懷好意的揣測，這也許是爲了讓服務器系統更有賣點）。說兼容，他自帶的兼容模式都不能很好的兼容很多程序，怎麼說好呢？說便捷，他也做不到像 Linux 那樣，敲一行命令就裝上一個軟件。很多地方都可以提升，但他不做，或做得好少就拿來賣。
M$ 大概是太看重 $ 了罷。產品尚未成熟，就急急忙忙的推出來換錢。 XP 之前有個 2000 ， Windows 7 之前有個 Vista ，都是市場不接受的不成熟產品。

爲什麼蘋果的市場份額會增加？我想，這和 M$ 的腐化與遲滯大有關聯。坐擁 90% 市場份額的 M$ 已經喪失了早期的活力和創意，升級帶有機械性，且是創收需求所致。 M$ 現在最富有的，就是龐大市場份額帶來的慣性吧。

後記
本文完成後半年多，本人的老爺機終於壽終正寢，於是買了配置不高的新電腦。由於一時腦殘，顯卡成了 A 卡的，需要 .Net 3.5 支持。 XP 自帶 .Net 2.0 ，但不帶 3.5 ，每次重裝都頗感不便。 M$ 故意把 .Net 4.5 設定成與 XP 不兼容，而我想玩的一個軟件需要之，就試了試 Windows 8 。果不其然， Windows 8 讓我各種不爽，於是趕快換成 Windows 7 。經過三年磨合， Windows 7 的軟件兼容性終於上了幾個臺階。於是我把任務欄設成跟 XP 一樣的，湊合用。最不方便的當然是 explorer 裏的感覺，由於無法設成和 XP 一樣，直接導致本人操作效率嚴重下降，而較多倚重 TotalCMD 了。
不巧，本文完成後一個多月， Office 2010 SP1 發佈，速度比 2007 大幅提升，比 2010 原版似乎也有提升，資源佔用也減小了很多。儘管冷啟動速度一般，熱啟動還不錯。再去掉一些無聊的加載項和信任中心選項，跟 2003 基本持平。 2013 相對於 2010 提升不大，但更加深度的整合 Skydrive 以及頗受爭議的 metro ，姑且算個進步。討厭的是普遍多了導航界面，對於高級用戶來說不方便。
可以說，本文恰好作於 M$ 的低谷，不過如今百花齊放， M$ 的份額被喫了不少
