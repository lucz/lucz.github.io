實體書

比較薄，而且網上沒有索引的工具書，實體書更好用。
大厚本用實體書很不方便。比如故訓彙纂，衹能放在桌子上翻，不能拿手裏。這樣要麼很佔位置，要麼拿來拿去容易散架。

工具書小結

中文有六大本：大漢和辭典、中文大辭典、漢語大詞典、漢語大字典、故訓彙纂、異體字字典。漢語大詞典有檢索版，但字詞不全。中文大辭典普及版、漢語大字典、故訓彙纂有人製作了查字程序。日本人做了漢語大詞典和大漢和辭典的詞條 txt ，但是可能是日本輸入法問題，有的字用頁數來替代了。異體字字典情況最好，但這本書恐怕衹有非常專業的人使用。

筆記

Evernote/Wiz

前者速度較快，後者支持 html ，更適合做深度排版。二者均可網上備份，但前者無法在無網絡的環境下使用。

OneNote

筆記整理的老大哥，最有紙質筆記的感覺。較適合深度整理。
如果有平板電腦或手寫版，手寫筆記很帶感。還可以摘抄書籍之後畫重點。

Marboo

沒有 Windows 版，但本人比較喜歡其工作方式——支持 rst 及 markdown 的筆記軟件，而且可以方便的跳到自己的文本編輯器。

Zim 等其他筆記軟件

可以說都有特色，但都不完善。

個人 wiki

普遍需要搭建 php 之類的動態環境，而且不能快速從瀏覽切換到文本編輯器編輯。
但是如果打算公開，倒是可以考慮——儘管我個人是覺得不如用 Jekyll 之類的靜態網站生成器。
本文可以看作一個個人 wiki 的一頁，由於使用了 Javascript 做索引，頁內跳轉還是比較方便的。

Visio

做思維導圖比較便捷，但其實思維導圖做起來都很麻煩。
亦有開源免費替代品，不過這些替代品實在太爛了。

硬件

鍵盤基礎知識

碼字的主要工具就是鍵盤，而且你還老摸他敲他，他眞的很重要對吧？
所以就講多一點。

鍵帽

ABS 容易打油。
PBT 一般做黑的以外各種顏色，不易打油。其極品爲熱昇華。
POM 一般做黑的，更不易打油。
金屬很貴，容易磕傷。
所謂打油就是手指把鍵帽磨得很光滑。

佈局

104 佈局是全鍵盤，包括方向鍵和數字小鍵盤。設計方爲 IBM 。
87 相當於 104 砍掉小鍵盤。
84 類似筆記本佈局。好處是如果旁邊放鼠標會很方便。
66-68 在 84 的基礎上再去掉 F1-F12 和一些鍵。
Unix: Control 的位置放到 CapsLock 。鍵很少。
MAC: 見於各種蘋果機。
純小鍵盤：專門輸入數字。
日韓鍵盤：空格左右各多一個特殊的鍵。
歐洲鍵盤：大回車，歐元符號，很多歐洲語言的鍵盤是非常詭異的。

字母佈局

QWERT: 乏善可陳，但是很多軟件的默認組合鍵都用他。
DVORAK: 比較適合英文輸入。
Programer’s DVORAK: 移動了標點符號的位置，以便編程。
COLEMAK: 考慮了組合鍵問題和舒適度。
其實雙拼及部分形碼已經和 QWERT 結合在一起了，所以這些用戶不建議換成別的佈局——不過可以考慮替換個別按鍵。

壓力

機械鍵盤一般都喜歡說這個，其實說克是不準確的，克是質量單位，而且機械鍵盤用的是彈簧，地球人都知道把彈簧壓縮一點點和壓扁用的力道不同。
一般而言，市面上說的 60g 算是一個分水嶺，大於這個的容易喫力。最小的有 30g 的，對某些人而言，手指搭上去就觸發了。

眞品購買法則

淘寶商城、大型電商不會有假貨。當然了，價格容易高一點。
一般仿品都是仿標準的中文或美版 104 ，所以如果是奇葩的歐版，基本上不會是假貨。
但是都知道歐版奇葩，有沒有不奇葩也不仿冒的便宜貨？有，那就是注音符號、泰文、希伯來文、各種奇葩文的 104 。他們的佈局和英文版一樣，但是上面有很多奇葩文字，價格一般介於歐版和美版之間。
另外，還有一個略奇葩的版本，就是韓版， 106 鍵，個人認爲比美版還好用。
最後，有些型號基本上沒什麼仿品，有些型號幾乎全是仿品……
機械鍵盤和靜電容鍵盤目前沒有仿品。

薄膜鍵盤

特點

價格便宜。
手感不夠穩定，一般也就堅持兩三年。經典的除外。
手感往往不出衆，但也有手感好的——當然這種好需要個人的體會。
分火山口和剪刀腳兩種，後者一般鍵程短，多見於筆記本鍵盤。
觸發壓力爲 60g 。
由於比較容易仿，所以名優產品的仿品很多。
現在的薄膜鍵盤不宜隨意清洗，因爲他的菊花那裏在出廠前會上油，清洗時會洗掉。不過也可以買一瓶雙飛燕鍵盤油塗抹一下。至於好不好，這就難說了——手感是細膩的東西。

機械鍵盤

特點

手感維持時間長。
手感好。鍵程長。
貴。
高，沒手托會比較累。
由於可玩性強又耐用，二手市場非常活躍。買鍵盤多的人出掉的僅拆貨，用兩年還能賣個好價。

Cherry 軸

德國產。一般而言買機械鍵盤就是買 MX 軸鍵盤。
MX 黑軸：壓力重，觸發短，無段落感，無聲，壽命長。不挑做工。
MX 紅軸：壓力最輕，觸發短，無段落感，無聲，壽命長。黑軸和紅軸如果不觸底，是沒有什麼聲音的。觸底時會有「噠」的一聲。
MX 茶軸：壓力輕，觸發短，輕微段落感，聲音小。早期的叫焦茶，現在穩定版是赤茶，前者更滑一些。
MX 青軸：壓力中，觸發長，明顯段落感，聲音大。配合鋼板很帶感，但是聲音也很可觀。
MX 白軸：壓力很重，觸發長，明顯段落感，聲音小，停產。有個特殊的版本叫脆白軸，據說特脆。有些人喜歡用白軸打字。
ML 軸：壓力輕，觸發和總行程都短，輕微段落感，聲音小，易卡鍵——洋垃圾很多。
六大軸體系下還有更多軸，比如灰軸是加重的黑軸，綠軸是加重的青軸，不過這兩個一般不作爲整個鍵盤使用，而是放在空格。另外還有很多稀有軸、停產軸。
所謂段落感，就是類似圓珠筆屁股的手感。有段落感的壽命都是 2000 萬次，而無段落感 5000 萬次。
一般認爲，青軸聲音清脆，打字帶感。茶軸和紅軸輕巧，打字輕鬆。黑軸除非力大，不適合打字。
青軸的觸發壓力跟薄膜鍵盤差不多，紅軸的觸發壓力比絕大多數薄膜鍵盤要輕盈，但觸底壓力還是比較大的，按 ctrl 等按鍵的時候仍然喫力。

ALPS 軸

原爲日產，後爲臺產和國產。市場認可度不如 Cherry 。
蘋果有些經典老鍵盤使用了日產的 ALPS 軸。
這類軸型號、產地極複雜，而且已經停產，舊貨也不好買。軸體有紅、橘紅、藍、白、黑、青、黃等。
小霸王學習機就是採用臺灣白軸的！

鋼板之有無

鋼板起到固定鍵軸的作用。鋼板會帶來較強的反饋，所以無鋼板的軟而鬆，有鋼板的硬而緊。
無鋼板鍵盤換軸比較方便，有鋼板的需要花幾十塊買電熱吸錫器和錫絲。
鋼板的換軸是一件比較值得注意的事——加熱不慎可導致焊盤脫落。
一般的非 Cherry 鍵盤都帶有鋼板，除了： KBC Poker 1, KBT Race, KBT Pure, Neo 87, Plum Morphling 96 的早期版本。因爲無鋼板需要五腳軸體，價格高，貨源緊。

靜電容鍵盤

特輕，觸發短，鍵程長，回彈好。
特貴。
耐用度中等。

鼠標

軌跡球可以有效治療鼠標手。
指環鼠標臨空指點也不累，就是不夠準確。
手寫板寫字美，也可以治療鼠標手。
ThinkPad 的指點杆可以讓手不離開鍵盤。
蘋果的 Magic Mouse 需要把鼠標放在桌子深處，然後把手臂伸直。這樣也能預防鼠標手。
普通鼠標，把椅子扶手墊高，也不會鼠標手。
精細作圖需要高精度鼠標或軌跡球。
太貴的鼠標對非遊戲玩家來說就是騙錢。
一個電腦熟手應該多學點快捷鍵，敲字時儘量不用碰鼠標。其實很多程序員並不怎麼用鼠標。
爲什麼我這麼在意鼠標手？因爲他眞的很痛痛痛痛痛……

顯示設備

看直排書，很多人喜歡用可以旋轉的屏幕，或者買個支架。
LED 和 LCD 都不算很傷眼，但仍然需要注意用眼衛生。
其實投影機更爽，但是流明度要四位數，不然不能開燈用。

主硬件

平板和手機衹能當玩具，長期用太損目。
出門讀書用漢王或 Kindle 。 Kindle 有國行，但是日行最便宜。
存書能存 3TB 已經很多了。花不了什麼錢。
ThinkPad 也許是最耐用的筆記本。
蘋果會導致大量軟件無效化，而且硬件成本其實不高。裝 Windows 的話，一來很搞笑，二來高發熱。
目前來講，電腦的 CPU 並非一個作者能使用完的，所以衹要用上較大的內存，和一塊不用多大的 SSD 固態硬盤，電腦的速度就可以維持在一個很好的水平上。

快捷鍵

寫作過程中，各種編輯類操作是免不了的。有快捷鍵肯定比沒有好。

SharpKeys

可以調換各種鍵位的軟件，使用非常簡便。支持多媒體鍵。
寫入註冊表，需要重啓。

AutoHotkey 與 AutoIt

開源軟件。
具備編程所需要素，可以當作編程語言來看待。
可以定義各種奇葩縮寫，各種奇葩快捷鍵，但是對多媒體按鍵之類的東西支持不好。
如果鍵盤上有打算廢物利用的多媒體按鍵，可以先通過 SharpKeys 定義爲 F13-F24 。

寫作軟件

輸入法

分類

音碼：根據字音輸入。重碼率高，但容易掌握。其中雙拼擊鍵量最小。
形碼：根據字形輸入。重碼率低，但要背字根，而且形碼的拆字邏輯都非常奇葩，能輕鬆接受形碼拆字邏輯的人文字學水平八成很差。
音形碼：以音爲主，加上開頭的一兩個形碼來減小重碼率。漢字的特點同音字如果形狀有相似部分，其不同部分往往是左、上部分，即手寫時先寫的部分。比如同爲金字旁，常用的銀、銅、鉄、鋁、錫、鉛讀音都不同。而同樣讀如「同」，銅、童、同、筒、桐、仝在偏旁上都是不同的。所以音形碼減小重碼率的效果還是比較好的。
手寫輸入：手寫即可，普遍需要逐字確認，慢，筆畫多的字會很煩。

發放形式

收費。除了嘸蝦米好像沒見有名的。
閉源免費。絕大多數。
開源免費。多爲 *nix 下的輸入法。

繁體字

以簡化字編碼，通過函數轉爲繁體：大部分中國輸入法的特色，錯字率很高。
臺灣繁體或香港繁體：當地輸入法，對我來說錯字率也不低。
中國繁體：貌似衹有 Rime 一個，不過理論上開源的輸入法都可以改過來。

文本標記語言

WYSIWYG ——所見即所得

Office: 如 Microsoft Office, LibreOffice, 上手易，適合處理簡單文件。
排版類：如方正飛騰、方正飛翔、 Adobe Indesign ，上手較難，進階更難，速度普遍較慢，格式特殊，但可以處理無規律複雜文檔，如海報。

TeX

古老而富有生命力的開源排版軟件。
可以進行非常複雜但有規律的排版，如棋譜、公式。
上手較難，處理東亞語系時比較喫力。

方正書版

很貴。
上手較容易，因爲官方文檔非常清晰。
可以看作 TeX 的東亞化基礎版，在處理東亞文字時比較輕鬆。

html+css

網頁排版。
標記非常囉嗦。
不同的瀏覽器會導致不同的瀏覽結果。
直排不給力。

markdown

兼容 html ，但有一些簡化的標記。
不過很多方面反人類。
很多網站開始支持了。

textile/rst

比 markdown 豐富且人性化。
流行度不及 markdown 。

文本編輯器

難點

多數文本編輯器是開發給程序員的，換言之，不適合一般寫作。
多數文本編輯器時開發給西洋人的，換言之，不適合東亞寫作。
有時候有些好東西我們衹能在 Windows 下乾瞪眼，因爲程序員用什麼系統的都有。

EmEditor

除了速度慢，以及沒有收藏夾，還有對 markdown 之類標記比較弱，可謂全能。

EditPlus

前者的劣化版，速度快，但略奇葩。

EverEdit

速度快， markdown 支持好，功能也相當豐富，積極吸取優秀編輯器的特點。
搜索和替換有問題，無收藏夾。

管理與備份

版本控制

所謂版本控制，就是由一個軟件來顯示你每次做了哪些修改。
版本控制在多人協作中優勢特別明顯。
但是版本控制一般用於開源的文件，二進制文件的差異太難找出了，更別說加密文件。

Git

由 Linus 開發的最人性化的版本控制軟件。
命令行操作，上手需要一點點時間。
也有圖形化控制版本，不過命令有所缺失。

SVN

比 Git 老，效果略差，但也不是沒有優勢。

備份型網盤

一般都收費，不過好在可以定時備份所有打算備份的文件。
版本差異會是非常頭痛的事情。
不推薦。

加密

加密有風險，你可能忘記密碼，或者丟失密鑰。
加密還會導致版本控制失效。
如果有寧可丟失，也不泄露的文件，應該使用加密——不過一般人衹要不學習陳冠希恐怕很少有這樣的文件。
軟件不開源就別用了，鬼知道有沒有後門呢。

TrueCrypt

類似虛擬光驅的生成一個虛擬的加密硬盤。
世界領先技術。可以用一個隨便的文件，比如一張圖，來作爲密鑰。

AxCrypt

加密單個文件用。
文件名也可以被加密。

壓縮軟件

僅支持密碼模式，而且不能加密文件名。

備份與存儲

單機備份

使用 Allway Sync 或 Good Sync 之類的備份軟件。
從頻繁操作的文件夾備份到備份文件夾，如移動硬盤。

FTP/WebDAV 備份

類似前者，但傳到遠程。

網盤備份

備份到網盤上。
一個知名網盤本身的備份一般做得比個人的要好。
但是網盤的協議往往有很多霸王條款，傳文件給他，他甚至有權做商業使用。
傳到中國網盤還可能被老大哥查看。

網盤同步

備份，是定時或不定時的傳到別處。同步則是實時的，多機同步還可能是從服務器到本地的。
很多網盤具有同步功能，甚至衹有同步功能，而無備份功能。
但是同步也是個雙刃劍，有時同步網盤會用已上傳的舊文件覆蓋本地的新文件。
最有名的服務是 Dropbox 實力強的還有 SkyDrive, Google Drive, iCloud 等大公司提供。但是普遍存在隱私問題。
SpiderOak 和 Wuala 提供安全可靠的隱私保護，用戶數據在服務器端無法瀏覽。

開源空間

可以在網絡上直接看版本控制的情況。
Github 對免費用戶強制開源，大小限制爲每個 1GB ，但是可以建立多個 repo ，曾經被牆，後恢復。
BitBucket 不強制開源，而且不限制大小，但是經常牆。
其實還有不少，不過名氣不那麼大。
